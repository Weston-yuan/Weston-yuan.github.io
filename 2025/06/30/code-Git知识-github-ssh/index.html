基本概念

仓库Repository：仓库用来存放代码，每个项目对应一个仓库，多个开源项目则有多个仓库。通过git管理仓库
收藏Star：收藏项目，方便下次查看
复制克隆项目Fork：复制别人的项目，(该Fork的项目是独立存在的）
发起请求Pull Request：
关注Watch：关注项目，当项目更新可以接到通知
事务卡片Issue：发现bug，待讨论

step2. 创建本地空仓库（.git）：git init     （用来存储该仓库所有信息）

2、向【工作区——暂存区——仓库】添加文件

step3.  将目标文件放入（本地工作区） cd到本地工作区

4、推送到GitHub： git push

使用GitHub远程仓库

目的：备份、代码集中共享

1、Git克隆远程仓库（GitHub的项目）
目的：将远程仓库（GitHub的项目）复制到本地。

step1. cd到本地目标文件夹（在文件夹内初始化Git，可选操作）

step2 git clone 仓库地址 （仓库地址在GitHub—项目—下载—地址）
            * 如果不对可以多来几次
           
2、向【工作区——暂存区——仓库】添加文件

step3.  将目标文件放入（本地工作区），cd到本地工作区
将目标文件（本地工作区）添加到暂存区：git add XXX

step4. 将目标文件从暂存区上传到本地仓库：git commit -m “…” （”内容说明”）
*查看当前状态：git status

3、将目标文件从【本地仓库】push到【远程仓库】
step5.  git push origin main / git push
*如果无法同步——将本地仓库中.git文件夹内config打开编辑
                [remote “origin”]
                url = https://github.com/Weston-yuan/ybeardesign.top.git
                #修改为
                [remote “origin”]
                url = https://用户名:密码@github.com/Weston-yuan/ybeardesign.top.git
将密码换成token

step6. 解决双分支
切换分支至master: git checkout master
强制重命名master分支为main分支: git branch main master -f
切换分支至main: git checkout main
强制推送本地main分支至远程库，并覆盖远程main分支内容: git push origin main -f
